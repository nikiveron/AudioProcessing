# 🎵 Sonara - Платформа для AI-обработки аудио

**Статус:** 🚧 Альфа-версия | В активной разработке  
**Назначение:** Полнофункциональная система для обработки аудиофайлов с использованием нейросетевой модели с поддержкой параметризации по жанру и инструменту

---

## 🧩 Описание проекта

**Sonara** — это микросервисная архитектура для асинхронной обработки аудиофайлов с применением машинного обучения. Система предоставляет веб-интерфейс для загрузки аудио, управления процессом обработки и скачивания результатов в реальном времени.

Проект объединяет:
- 🎨 **Современный фронтенд** (Vue 3) для взаимодействия пользователя
- ⚡ **ASP.NET Core WebAPI** для управления бизнес-логикой
- 🤖 **ML-сервис** (Python) для нейросетевой обработки аудио
- 📨 **Event-driven архитектуру** на базе Apache Kafka для асинхронной обработки

---

## 🔍 Основной функционал

### 👨‍💼 Фронтенд (Vue 3 + TypeScript)
- Загрузка аудиофайлов через веб-интерфейс
- Выбор параметров обработки:
  - **Инструменты:** гитара, пианино, вокал, барабаны и другое
- Real-time отслеживание статуса обработки (SignalR WebSocket)
- Скачивание обработанных аудиофайлов
- Встроенный аудио-плеер для проигрывания результатов

### 🔧 Бекенд (ASP.NET Core 8)
- REST API для управления треками и заданиями
- Интеграция с MinIO для хранения файлов
- Управление жизненным циклом задач 
- SignalR Hub для push-уведомлений о статусе
- PostgreSQL база данных для хранения метаданных
- Kafka Producer для создания заданий обработки

### 🤖 ML-сервис (Python)
- Асинхронная обработка аудио с использованием нейросетевой модели GRU
- Параметризованная обработка (жанры, инструменты)
- Kafka Consumer для получения заданий
- Скачивание входных файлов из MinIO
- Загрузка результатов обработки обратно в MinIO
- Health checks и мониторинг состояния сервиса

### 👷 Worker-сервис (C# .NET)
- Orchestration логика для обработки заданий
- Координация между фронтенд, бекенд и ML-сервисом
- Управление Kafka топиками и сообщениями

---

### 📊 Workflow обработки файла:
1. **Загрузка** → Пользователь загружает аудиофайл через фронтенд
2. **Сохранение** → Файл сохраняется в MinIO
3. **Регистрация** → Создается Track запись в БД с InputKey и OutputKey
4. **Подписка** → Фронтенд подписывается на обновления через SignalR
5. **Обработка** → Бекенд отправляет задание в Kafka, ML-сервис обрабатывает
6. **Уведомление** → После завершения, JobStatusConsumer отправляет результат через SignalR
7. **Скачивание** → Пользователь скачивает обработанный файл

## 📦 Стек технологий

### Бекенд
- **Язык:** C# 
- **Платформа:** ASP.NET Core 8
- **API:** REST + SignalR (WebSocket)
- **База данных:** PostgreSQL 15
- **ORM:** Entity Framework Core
- **Message Broker:** Apache Kafka 7.5
- **Хранилище файлов:** MinIO (S3-compatible)

### Фронтенд
- **Язык:** TypeScript
- **Фреймворк:** Vue 3
- **Сборка:** Vite
- **HTTP клиент:** Axios
- **Real-time:** @microsoft/signalr
- **Контейнер:** Nginx + Node.js

### ML-сервис
- **Язык:** Python 3.10+
- **Фреймворк:** FastAPI
- **Deep Learning:** PyTorch
- **Модель:** GRU-based Audio Processor
- **Message Consumer:** Confluent Kafka

### Инфраструктура
- **Контейнеризация:** Docker & Docker Compose
- **Мониторинг Kafka:** Kafka UI
- **Управление БД:** pgAdmin 4
- **Оркестрация:** Docker Compose (локально)
